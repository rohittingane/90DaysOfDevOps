# My 90 Days DevOps Learning Plan

## Current Level

I am currently working as a Production Support Analyst. While working in support, I became interested in how applications are deployed, monitored, and maintained in production environments. This motivated me to start learning DevOps.

## What DevOps Means to Me

According to me, DevOps is not just about tools. It is about improving collaboration, automating repetitive tasks, and delivering applications faster and more reliably.

Cloud platforms help organizations run applications without managing physical infrastructure and provide flexibility and scalability.

## Why I Am Starting This Journey

My goal is to move into a DevOps role and build practical skills that are used in real projects. I want to understand not only how applications run but also how the complete infrastructure and deployment process works.

## My Goals for the Next 90 Days

1. Improve my Linux, Git, and AWS skills through daily practice.
2. Learn Docker, Jenkins, Terraform, and Kubernetes with hands-on labs.
3. Build and publish DevOps projects on GitHub to showcase my learning.

## Core Skills I Want To Build

* Linux troubleshooting
* CI/CD pipeline automation
* Kubernetes and container management

## Weekly Time Commitment

* Weekdays: 2–3 hours daily
* Weekends: 4–6 hours daily

## How I Will Stay Consistent

I will dedicate time every day to learning and practicing DevOps. I will maintain notes, complete assignments, push my work to GitHub, and share my progress on LinkedIn whenever possible.

## My Target

At the end of these 90 days, I want to be confident enough to apply for DevOps Engineer opportunities and demonstrate my skills through hands-on projects and continuous learning.
